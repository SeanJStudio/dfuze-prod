    Run a Job

  1. Click File ➜ New Job or press Space to show the Job window.

  2. Make a selection from the Client dropdown, then another selection from the Job dropdown. Once a Job is chosen you will see information displayed on the right.

  3. Click the "Load Data" button to import your data. The location of the last file Dfuze interacted with will open for ease of use. Hold CTRL or SHIFT to merge multiple files. The file types .accdb, .cert-1, .csv, .dat, .dbf, .mdb, .txt, .xls, .xlsm, and .xlsx are supported. Text based files will be automatically encoded and any BOM markers will be removed.

  4. Next you must map any required fields from the data to the required fields. Dfuze automatically maps most required fields but you should verify it was done correctly, pressing right/left on your keyboard to preview record data can help with this. If fields arent automatically mapped you must drag the fields on the left to the required fields on the right.

    You can merge fields together into a new field by holding CTRL and selecting multiple fields on the left and pressing the "Merge New Field" button.

    Checking the "Remember fields used for this job" box will save the fields used for future use if available.

  5. Once mapped, click Job ➜ Run or press R to run the Job.

    Export Database

  • After a Job has run click Data ➜ Export or press E to show the export window.

  • Data can be exported as .accdb, .csv, .dat, .dbf, .mdb, .txt, or .xlsx file types.

  • Export multiple data files based on unique values within a field. Ex. if the field "Fruit" is selected and has the values "apple, orange, banana" then 3 files will be created.

  • Press the "Export Data" button to browse for the location to export data.

    Dedupe Names & Addresses

    1. Click File ➜ New Job then select "0 - Utility" as the Client and "Dedupe" as the Job.

    2. After mapping the required fields and running the job, the dedupe window will display. Here you can customize how matches are made, choose your name fields, prioritize records, view/modify/export/delete dupes.

    3. After choosing your settings click run to start. The dedupe works by normalizing the address fields by fixing errors, converting accents, converting numbers to words, changing spelling of hundreds of components to be the same in multiple languages, parsing out various components, removing vowels and comparing the final parts to find enough of a match.

      When a name is selected it is compared by removing hundreds of prefixes/suffixes, converting accents, parsing multiple names into different components, going through 4 string matching algorithms (Jaro Wrinkler, NYSIIS, Metaphone3, Refined Soundex), comparing initials, comparing reversed names and more!

    4. Once the dedupe finishes its a good idea to click the view button to view the dupes and make any necessary modifications. If you want to remove the dupes from the database, then click the delete button.

    5. You can run the dedupe again with different settings or if you're finished close the window and click Data ➜ Export or press E to show the export window to export your database.

    Disparate Data Normalizer (Rental)

    1. Click File ➜ New Job then select "0 - Utility" as the Client and "Rental" as the Job.

    2. After running the job Dfuze will attempt to map the headers from the imported files to a set of common predefined headers and then displays the mapping for you to review. If everything looks good, close the window to continue.

    3. Dfuze will then analyze the name fields to fix data entry errors and identify companies.

    4. Next it will prompt you to select how the salutation field should be built.

    5. Once finished, click Data ➜ Export or press E to show the export window to export your database with the normalized fields appended.

    Regex Search Database

    1. After running any job you can click Data ➜ View or press V to view the database.

    2. Here you can type a regex pattern that will be searched against the entire database with only the matched records returned and their matched parts highlighted in yellow.

    3. If you are unfamiliar with regex patterns, check out this site.

    4. If you want to export the data, select the rows you want and click "Export Selected Rows"

    Tools

  • After running a Job, various tools become available in the "Tools" section.

  1. Casing Conversion - Variety of options including auto casing

  2. Encoding Correction - Fix incorrectly encoded characters

  3. Inkjet Maker - Batch create inkjet printing files and layout report

  4. Proof Maker - Create a proof database for client review

  5. Sample Maker - Make a sample file for internal tracking
